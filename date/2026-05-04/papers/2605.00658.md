# UniVidX: A Unified Multimodal Framework for Versatile Video Generation via Diffusion Priors

**【统一框架多模态视频生成】**

**arXiv**: https://arxiv.org/abs/2605.00658

## 简要摘要

*计算机视觉；视频生成；多模态扩散模型*

现有视频扩散模型通常针对单一任务单独训练，限制了跨模态关联建模。
提出统一框架UniVidX，通过随机条件掩码、解耦门控LoRA和跨模态注意力设计，实现多种模态间的视频生成与编辑。

## 深度解析

这篇论文《UniVidX: A Unified Multimodal Framework for Versatile Video Generation via Diffusion Priors》是一篇关于**视频多模态生成**的研究。下面我用简单的语言为你梳理它的核心内容：

---

### (1) 论文所在的任务是什么？
- **任务输入**：可以是**文字描述（Text）**或**一种/多种视觉模态**（例如：RGB视频、反照率图Albedo、法线图Normal、透明度通道Alpha等）。
- **任务输出**：生成**另一种或多种视觉模态**（例如：根据文字生成视频、根据RGB视频生成法线图、根据反照率生成光照效果等）。
- **任务功能**：实现**文字到视觉、视觉到视觉、文字+视觉到视觉**等多种视频生成任务，例如：
  - 文字生成视频（Text → Video）
  - 视频分解（Video → Albedo/Normal/Alpha）
  - 视频重光照（Relighting）、重贴图（Retexturing）、背景替换等。

---

### (2) 论文想解决什么问题？
现有的视频扩散模型（Video Diffusion Models）虽然强大，但通常**一个模型只能做一种任务**（例如只能从文字生成视频，或只能从视频生成法线图）。这导致：
1. **灵活性差**：无法根据不同的输入条件灵活切换任务。
2. **忽略模态关联**：不同视觉模态（如颜色、光照、形状）之间本应有关联，但分开训练会导致生成结果不一致。
3. **数据效率低**：每个任务都需要大量标注数据训练。

本文的目标是：**设计一个统一的框架，让一个模型能处理多种输入输出组合，并保持跨模态的一致性**。

---

### (3) 核心思路是什么？
作者提出三个关键设计：

1. **随机条件掩码（Stochastic Condition Masking, SCM）**  
   训练时随机将某些模态作为“干净条件”，某些作为“带噪声的目标”，让模型学会**任意方向的条件生成**，而不是固定输入输出。

2. **解耦门控LoRA（Decoupled Gated LoRA, DGL）**  
   为每个视觉模态附加一个轻量化的LoRA模块，只在生成该模态时激活。这样既能适应不同模态的分布，又**不破坏基础模型的原有能力**。

3. **跨模态自注意力（Cross-Modal Self-Attention, CMSA）**  
   让不同模态在注意力层共享信息（共享Key/Value），但保持各自的查询（Query），促进**模态间的对齐与信息交换**。

---

### (4) 结果怎么样？
- **性能优越**：在多个任务（文字生成、反照率估计、法线估计、视频抠图等）上达到或接近当前最优方法。
- **泛化能力强**：即使只用**不到1000个视频**训练，也能在真实场景中表现良好，说明框架**数据效率高**。
- **支持多种应用**：如视频重光照、材质编辑、背景替换等，展示了框架的实用性。

---

### (5) 模型结构与流程
- **基础模型（基模）**：基于**预训练的视频扩散模型（VDM）**，例如 Stable Video Diffusion 或类似结构。
- **整体流程**：
  1. **输入阶段**：接受多模态输入（文字 + 一种或多种视觉模态）。
  2. **编码与融合**：
     - 文字通过CLIP等文本编码器得到文本特征。
     - 视觉模态通过各自的编码器（如CNN或ViT）提取特征。
     - 通过**跨模态自注意力（CMSA）**将不同模态的特征进行交互对齐。
  3. **条件处理**：
     - 训练时使用**随机条件掩码（SCM）**，随机选择一部分模态作为条件（保持干净），另一部分作为生成目标（添加噪声）。
     - 推理时根据任务需求指定条件模态。
  4. **生成阶段**：
     - 扩散模型逐步去噪声生成目标模态。
     - **解耦门控LoRA（DGL）**根据当前生成的目标模态激活对应的LoRA模块，调整生成过程。
  5. **输出**：得到目标模态的视频序列（如RGB视频、法线图、Alpha通道等）。

**流程图示意**：  
```
多模态输入 → 编码 → CMSA跨模态交互 → SCM条件选择 → 扩散去噪 + DGL模态适配 → 输出目标模态
```

---

### 总结
这篇论文的核心贡献是提出了一个**统一的多模态视频生成框架**，通过**随机条件掩码、解耦LoRA、跨模态注意力**三个设计，让一个模型能灵活处理多种视频生成任务，并保持跨模态一致性，同时数据效率很高。
