# UniVidX: A Unified Multimodal Framework for Versatile Video Generation via Diffusion Priors

**【多模态视频统一生成框架】**

**arXiv**: https://arxiv.org/abs/2605.00658  
**AlphaXiv**: https://www.alphaxiv.org/zh/overview/2605.00658  
**HF Paper**: https://hf-mirror.com/papers/2605.00658

## 简要摘要

*计算机图形学;视频生成;多模态扩散模型*

现有视频扩散模型常针对不同任务分别训练，导致输入输出固定且模态关联性弱
提出UniVidX框架，通过条件掩码、解耦适配器与跨模态注意力实现多模态视频统一生成

## 深度解析

好的，我们来用简单的语言梳理一下这篇论文。你可以把它看作是一个“视频版的多功能生成器”的说明书。

### (1) 这篇论文所在的任务是什么？
*   **任务输入**：**非常灵活**。可以是文字描述，也可以是视频的各种“成分图”（比如物体的固有色图、光照图、法线图、前景/背景分离图等），或者是它们的任意组合。
*   **任务输出**：同样灵活，可以生成你想要的**任何一种视频成分图**。比如，输入文字“一个发光的球”，可以生成这个球的法线图；输入一张真实视频和它的法线图，可以生成它的固有色图；输入一张前景透明图，可以补全它的背景视频。
*   **任务功能**：构建一个**统一的多模态视频生成框架**。它不像传统方法那样，一个模型只能干一件事（比如只能从文字生成法线图）。这个框架可以像一个“瑞士军刀”，根据你给的不同输入条件，自动切换到对应的生成模式，输出你想要的视频成分，并且保证不同成分之间是协调一致的。

简单比喻：传统方法是造了多个**专用工具**（螺丝刀只拧螺丝，锤子只敲钉子）。这篇论文想造一个**多功能电动工具**，换不同的头（输入条件）就能干不同的活（输出不同的视频成分）。

### (2) 这篇论文想解决什么问题？
主要解决现有方法中的两个核心痛点：
1.  **不够灵活**：一个模型只能处理一种固定的输入输出组合（例如，只能“文字→法线图”）。当任务需求变化时，需要重新训练一个新模型，效率很低。
2.  **缺乏一致性**：当需要生成多个相关联的视频成分（比如同时生成固有色、光照和法线图）时，现有方法往往是分开生成或者依次生成。这容易导致最终合成的结果中，各个成分之间对不上（比如光影和物体形状不匹配）。

所以，论文的核心目标是：**让一个视频扩散模型学会“多任务处理”，能根据任意给定的条件（文字或某些视频成分），灵活且一致地生成其他缺失的视频成分。**

### (3) 论文的核心思路是什么？
核心思路是**用一套统一的训练方法，教一个预训练好的视频生成模型（基模）学会“看图说话”和“补全图片”**。为了实现这个目标，他们提出了三个关键技术：

1.  **随机条件掩码**：在训练时，**随机**把一些视频成分（如固有色图）当作清晰的“条件”输入给模型，把另一些成分（如法线图）加噪后当作需要预测的“目标”。这样，模型就不知道下次训练会接到什么任务，被迫学会处理所有可能的条件-目标组合，从而变得非常灵活。
2.  **解耦门控LoRA**：为了在不破坏基模强大生成能力的前提下，让它学会处理新类型的输入（各种成分图），他们在模型里为**每种成分**都附加了一个小的、可训练的适配模块。只有当某种成分作为“生成目标”时，对应的适配模块才会被激活。这就像给基模配了多个可插拔的“技能卡”，需要哪个用哪个，不影响基模本身的核心能力。
3.  **跨模态自注意力**：为了让模型生成的各个成分之间保持协调一致（比如物体的边缘在所有成分图中都对齐），他们在注意力机制上做了改进。让不同成分的特征在计算注意力时**共享关键信息**，但保留各自的**查询**。这样，不同成分之间可以充分交流信息，确保最终结果的一致性和合理性。

### (4) 结果怎么样？
论文展示了两个具体的模型实例：
*   **UniVid-Intrinsic**：处理视频的“本质属性”（固有色、光照、法线）。
*   **UniVid-Alpha**：处理视频的前景、背景分离。

**效果很好**：
*   **性能有竞争力**：在多个单项任务上（如从文字生成法线图、视频抠图等），效果和当前最好的专用模型相当。
*   **泛化能力强**：即使在很小的数据集（<1000个视频）上训练，也能很好地处理真实世界中的复杂视频。
*   **应用广泛**：展示了视频重打光、材质编辑、视频修复、背景替换等多种实际应用，效果自然且一致。

### (5) 论文的模型结构
*   **基模**：论文使用的是现成的、强大的**视频扩散模型**作为基础。你可以把它想象成一个已经看过海量视频、对真实世界运动规律有深刻理解的“大脑”。
*   **整体流程**：
    1.  **输入准备**：用户提供输入，这可能是**文字**，也可能是**一个或多个视频成分图**（比如给了固有色图和法线图），或者是**文字+成分图**的组合。
    2.  **条件编码**：文字会通过一个文本编码器变成特征。每个视频成分图也会通过一个专门的图像编码器（例如 CLIP 的图像编码器）变成特征。
    3.  **随机掩码（仅训练时）**：在训练阶段，系统会**随机决定**哪些成分特征是清晰的“条件”，哪些成分需要被加噪并作为“预测目标”。这一步是模型学会灵活多变的关键。
    4.  **特征融合与处理**：所有条件的特征（文本的、图像的）被拼接起来，输入到**核心的UNet扩散模型**中。这个UNet内部集成了论文提出的三个关键技术：
        *   使用**跨模态自注意力**让不同成分的特征互相沟通。
        *   根据当前要预测的目标成分，动态激活对应的**解耦门控LoRA**模块。
    5.  **去噪生成**：UNet根据给定的条件，一步步对噪声进行预测和去除，最终还原出清晰的目标视频成分图。
    6.  **输出**：得到用户想要生成的视频成分，例如一段新的法线图视频，或者一个抠好的前景透明视频。

**总结一下流程**：用户**灵活指定**条件和目标 → 系统**编码**所有输入 →（训练时随机化任务）→ 核心UNet在**特殊注意力机制和适配模块**的帮助下，根据条件进行**去噪生成** → **输出**目标视频成分。整个过程由一个模型统一完成，实现了“一专多能”。
